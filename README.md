# [PARO2026] C++ Standard Template Library
W zadaniach zmieniamy ciała funkcji. Interfejsów nie ruszamy. Testy powinny przechodzić po wykonaniu zadań.

Kompilacja:

```bash
# Clone repository + setup build environment
git clone git@github.com:sz3kz/PARO2026-CPP-Standard-Template-Library.git
cd PARO2026-CPP-Standard-Template-Library/
make init

# Build & Execute binaries (run tests)
make build
make run

# Package solution for evaluation
make package
```

Po skompilowaniu powstają 3 pliki binarne odpalające testy GTest: pow2, cheap, unique.
